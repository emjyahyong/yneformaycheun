Spring Boot 3.4 : le support natif des HTTP Interfaces

Depuis la version 3.4, Spring Boot embarque un support natif pour les HTTP Interfaces : une simple interface Java annotée suffit à décrire un client HTTP, sans écrire la moindre implémentation. Le framework génère le client au démarrage à partir des annotations et d'un RestClient ou WebClient sous-jacent.

Pour un projet comme Yneformaycheun, où le FetchService doit interroger des flux RSS hétérogènes, cette approche ne remplace pas la librairie ROME côté parsing — mais elle simplifie nettement les appels vers des API annexes (vérification d'URL, résolution de redirections) en évitant le code de plomberie habituel autour de RestTemplate.

Le point de vigilance reste la gestion des erreurs : sans configuration explicite d'un ResponseErrorHandler, une réponse 4xx ou 5xx lève une exception générique peu exploitable — à valider avant d'introduire ce pattern sur un service qui doit rester tolérant aux flux mal formés.
